# yamme parameter sweep

Empirical sweep used to pick the compile-time tunables in `src/yamme.c`
(`TOP_K_SEEDS`, `REFINE_PASSES`, `MIN_REFINE_HITS`, `HAMMING_MISMATCH`,
`MIN_IC_BITS`).  The harness is kept in-tree so the same sweep can be
re-run when algorithm changes might shift the optimum.

## Result of the original sweep

| Constant | Old default | New default | Source |
|---|---|---|---|
| `TOP_K_SEEDS` | 20 | **4** | main 4x4 grid |
| `REFINE_PASSES` | 2 | **2** | main 4x4 grid (no improvement past 2) |
| `MIN_REFINE_HITS` | 10 | **20** | follow-up one-axis sweep |
| `HAMMING_MISMATCH` | 1 | **1** | follow-up one-axis sweep |
| `MIN_IC_BITS` | — | **0.5** | added separately for output-time trimming |

These values give 6/6 top-1 recovery on the specific-motif grid, identical
recall and PWM fidelity to old defaults on degenerate motifs (verified
across 20 replicated and stress-test fixtures), and ~50% fewer noise
motifs / ~30% less runtime than the old defaults.

## Prerequisites

- A C compiler with GNU99 support, GNU Make, zlib (same as building yamtk).
- Python 3 for fixture generation and scoring.
- (optional, for `run_full_verify.sh`) STREME from the MEME suite at
  `~/meme/bin/streme`.  Adjust the `STREME_BIN` variable in the script
  if installed elsewhere.

## Workflow

```sh
# 1. Generate all fixtures (deterministic; gitignored).
python3 gen_fixtures.py

# 2. Main 4x4 grid: TOP_K_SEEDS x REFINE_PASSES across D1-D6.
bash run_sweep.sh          # 16 builds + 96 yamme runs, ~12 min
python3 score.py           # writes results/summary.md (main grid section)

# 3. Follow-up one-axis sweeps at the main-grid winner: HAMMING_MISMATCH=2,
#    MIN_REFINE_HITS in {5, 20}, all at the chosen (k, r).
bash run_followup.sh       # ~1 min
python3 score_followup.py  # appends a section to results/summary.md

# 4. Multi-motif check on D7, D8 (and degenerate D9-D11) at three candidate
#    presets (current default, recommended, hedge).
bash run_multimotif.sh
python3 score_multimotif.py

# 5. Degenerate-preset verification: D9-D11 only, sweeping a few candidate
#    "ambiguous mode" presets to confirm whether a separate -A flag is needed.
bash run_degen_verify.sh
python3 score_degen_verify.py

# 6. Full verification: 5 replicates of D9-D11, big variants D12-D14,
#    IUPAC stress tests D15-D16, comparing yamme vs STREME per dataset.
bash run_full_verify.sh    # ~10 min
python3 score_full_verify.py
```

Each script appends its findings to `results/summary.md`.

## Selection rule (main grid)

A grid point is **feasible** if rank-1 recovery succeeds on all 6 datasets
(D1–D6).  Among feasible points the recommendation minimizes total noise
motifs, then total runtime, then maximizes mean `-log10 p` of the
recovered motif.  See `score.py` for the implementation.

## Layout

```
bench/yamme_sweep/
├── gen_fixtures.py            # synthetic dataset generator (deterministic)
├── run_sweep.sh               # main 4x4 (TOP_K_SEEDS x REFINE_PASSES) driver
├── run_followup.sh            # HAMMING_MISMATCH & MIN_REFINE_HITS one-axis sweeps
├── run_multimotif.sh          # multi-motif D7/D8 + degen D9-D11 at 3 presets
├── run_degen_verify.sh        # degen-preset candidates on D9-D11
├── run_full_verify.sh         # full degen verification + per-dataset STREME
├── score.py                   # score main grid; defines parse helpers reused below
├── score_followup.py          # score follow-up one-axis sweeps
├── score_multimotif.py        # score multi-motif results
├── score_degenerate.py        # IUPAC-aware match + PWM fidelity helpers
├── score_degen_verify.py      # score degen-preset candidates
├── score_full_verify.py       # score full verification (yamme presets vs STREME)
├── README.md                  # this file
├── fixtures/                  # generated FASTAs (gitignored) + planted.tsv
└── results/                   # gitignored
    ├── builds/                # per-preset yamtk binaries + .o files
    ├── tsv/                   # per-(dataset, preset) yamme TSV + MEME output
    ├── streme/                # per-dataset STREME output directories
    ├── timings*.tsv           # wall-clock per run, one file per sweep stage
    └── summary.md             # human-readable summary, appended per stage
```
